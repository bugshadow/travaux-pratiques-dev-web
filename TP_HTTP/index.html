TP 2 : Maîtrise de cURL

Question 2.1

Quelle est la difference entre -i et -v ?
-i (include) inclut uniquement les en-têtes HTTP de la reponse dans la sortie en plus du corps.
-v (verbose) est le mode de debogage. Il affiche toute la chronologie (resolution DNS, handshake TCP/TLS, en-têtes de requête envoyes (avec >), en-têtes de reponse reçus (avec <) et d'autres informations de contexte.

Exercice avance

Commande cURL demandee :

curl -i -X POST \
  -H "Content-Type: application/json" \
  -H "X-Custom-Header: MonHeader" \
  -d '{"action": "test", "value": 42}' \
  https://httpbin.org/post

Explication : -i permet d'afficher les en-têtes de la reponse de maniere lisible. -X POST n'est pas strictement necessaire ici vu qu'on utilise -d, mais c'est bien de le preciser.

TP 3 : API REST avec JavaScript

Exercice pratique : fetchWithRetry

Fonction avec tentatives multiples en cas d'erreur serveur :

async function fetchWithRetry(url, options = {}, maxRetries = 3) {
    for (let i = 0; i <= maxRetries; i++) {
        try {
            const response = await fetch(url, options);
            
            // Si le code est 5xx (Erreur serveur), on leve une erreur pour declencher le catch
            if (response.status >= 500 && response.status < 600) {
                throw new Error(`Erreur serveur: ${response.status}`);
            }
            
            // Sinon on retourne la reponse (même les 4xx)
            return response;
            
        } catch (error) {
            console.warn(`Tentative ${i + 1} echouee :`, error.message);
            
            // Si c'etait la derniere tentative, on propage l'erreur
            if (i === maxRetries) {
                throw error;
            }
            
            // Sinon on attend 1 seconde avant la prochaine boucle
            await new Promise(resolve => setTimeout(resolve, 1000));
        }
    }
}

Exercice 2 : Questions theoriques

  • Difference entre no-cache et no-store :
    no-store indique au navigateur/proxy de ne JAMAIS stocker la ressource. no-cache permet l'enregistrement, mais oblige le navigateur a valider avec le serveur (via ETag/If-None-Match) avant de l'utiliser.
  • Pourquoi POST n'est-il pas idempotent ?
    Une methode idempotente donne le même resultat sur le serveur si elle est executee une ou plusieurs fois (ex: GET, PUT). Repeter un POST va generalement creer autant de nouvelles ressources (ex: 3 clics sur "Acheter" = 3 commandes creees).
  • Que se passe-t-il si le serveur renvoie 301 ?
    C'est une redirection permanente. Le navigateur enregistre en cache que l'ancienne URL pointe desormais vers la nouvelle, et il effectue une nouvelle requête automatiquement vers la nouvelle adresse ("Location").
  • a quoi sert le header Origin ?
    Il indique d'où vient la requête (Protocole + Domaine + Port). Il est principalement utilise par le serveur pour autoriser ou bloquer les requêtes Cross-Origin (CORS).
  • Pourquoi utiliser HttpOnly sur les cookies de session ?
    Cela empêche le code JavaScript (ex: une attaque XSS reussie) de lire la valeur du cookie (comme le document.cookie). Le navigateur l'envoie de maniere transparente uniquement lors des requêtes HTTP.
